# АС - авто сотрудникам

Telegram-бот для контроля выдачи служебных авто сотрудникам в личное пользование.

## Что умеет MVP

- Распознает сообщения о выдаче:
  - `Взял JAC J7 Н537РА126 на 2 дня, возврат 29 августа`
  - `Взял JAC J7 Н537РА126 на 1 день, ночная смена, возврат завтра`
- Распознает возврат:
  - `Вернул JAC J7 Н537РА126, аренда 2 дня`
- Дает сотруднику кнопки `Взять машину` и `Сдать машину`.
- Марку авто сотрудник выбирает из списка: `КОМФОРТ +`, `КОМФОРТ`, `ЭКОНОМ`.
- При выдаче просит 5 фото: 4 стороны авто и приборная панель с уровнем топлива. Комментарий при выдаче можно пропустить.
- При сдаче требует комментарий о недостатках или кнопку `Машина полностью исправна`.
- При сдаче просит 5 фото: 4 стороны авто и приборная панель с уровнем топлива.
- Если фото меньше нормы, бот не фиксирует выдачу или возврат и не дает завершить сценарий.
- Фото не скачиваются на сервер: бот отправляет их компактным альбомом в рабочий чат `WORK_CHAT_ID`.
- Хранит выдачи, возвраты и оплату в SQLite.
- Формирует Excel-отчет по запросу: ФИО сотрудника, username, госномер, марка авто, даты аренды, количество дней, комментарии при взятии и сдаче.
- В кнопочном сценарии сотрудник указывает только дату/время возврата, количество дней бот считает автоматически.
- Сортирует отчет по ФИО сотрудника, а внутри сотрудника - по количеству дней аренды от большего к меньшему.
- Умеет выгружать отчет по отдельному сотруднику.
- Умеет отмечать списание аренды командой `/paid` в ответ на сообщение о возврате.
- Отправляет предупреждения, если формат нарушен или не хватает фото.

## Запуск на сервере

```bash
cp .env.example .env
```

Заполните `.env`:

```bash
BOT_TOKEN=123456:telegram-token
WORK_CHAT_ID=-1001234567890
APPROVER_USERNAMES=sergey,stax_ru
ADMIN_USERNAMES=your_username,stax_ru
FREE_MODELS=JAC J7:10
DRIVER_RATES=JAC J7:2400
EXEMPT_USERNAMES=hussein,sasha_f
DB_PATH=/app/data/car_rental_bot.sqlite3
```

Запуск:

```bash
docker compose up -d --build
```

Проверка логов:

```bash
docker compose logs -f autostaxbot
```

## Важные ограничения Telegram

Telegram присылает фото альбомом как несколько сообщений с одним `media_group_id`.
Бот собирает такие сообщения в течение нескольких секунд и считает количество фото. В кнопочном сценарии бот ждет все фото по одному и не завершает заявку, пока не получит полный комплект.

Фото не хранятся локально. Бот отправляет их альбомом в рабочий чат из переменной `WORK_CHAT_ID` вместе с короткой сводкой по выдаче или возврату.

Реакции Telegram не всегда доступны ботам в обычных группах, поэтому MVP поддерживает надежную команду:

```text
/paid
```

Ее нужно отправить ответом на сообщение `Вернул ...`. Если реакции доступны в вашем чате, обработчик `✅` можно добавить следующим шагом.

## Команды

- `/start` - открыть верхнее меню бота. У обычных пользователей будут кнопки `Взять машину` и `Сдать машину`; у пользователей из `REPORT_USERNAMES` дополнительно появятся кнопки отчетов.
- `/status` - краткая статистика активных аренд.
- `/car` - показать кнопки выдачи и сдачи.
- `/chat_id` - показать ID текущего чата для настройки `WORK_CHAT_ID`.
- `/set_work_chat` - назначить текущий чат рабочим для пересылки фото и сводок.
- `/report_month` - отчет за текущий месяц.
- `/report_month 2026-05` - отчет за указанный месяц.
- `/report_employee 2026-05 @username` - отчет за месяц по сотруднику.
- `/report_employee Иван Иванов` - отчет за текущий месяц по ФИО.
- `/employees 2026-05` - список сотрудников, по которым можно выгрузить отдельный отчет.
- `/paid` - отметить списание аренды, команду нужно отправлять reply на возврат.
- `/help` - показать форматы сообщений.

## Следующий шаг

После добавления бота в реальный чат нужно:

1. Выдать боту права читать сообщения.
2. Отключить privacy mode у бота через BotFather, иначе он не увидит обычные сообщения в группе.
3. Добавить в `.env` Telegram usernames администраторов, `REPORT_USERNAMES` для отчетов и `WORK_CHAT_ID` рабочего чата.
4. Перезапустить бота: `docker compose up -d --build`. При старте он закрепит `/start` в меню команд Telegram.
